# 🎙️ Audio-to-Note App: Full-Stack Architecture

A hyper-modern React Native mobile application designed to capture live auditory lectures and seamlessly convert them into perfectly formatted, intelligently categorized markdown study notes using **Google's Gemini 2.5 Flash** multimodal engine.

---

## ⚙️ Tech Stack

### Frontend (Mobile App)
* **Framework:** Expo (SDK 54) & React Native
* **Routing:** Expo Router (`app/` directory structure)
* **Styling:** NativeWind (Tailwind CSS for React Native) & Custom Dark/Light Theme Engine
* **Hardware Modules:** `expo-av` (Microphone access), `expo-file-system` (Persistence mapping)
* **AI Provider:** `@google/generative-ai` (Gemini SDK)
* **Rendering:** `react-native-markdown-display` (Native Markdown parsing)
* **State Animations:** `react-native-reanimated`

### Backend (API Server)
* **Framework:** Express.js (TypeScript)
* **Database ORM:** Prisma v6
* **Database Engine:** Remote PostgreSQL (`db.prisma.io`)
* **Environment Configuration:** `dotenv`

## 🏗️ Core Architecture & Data Flow

### 1. Audio Capture & Persistence Pipeline (`app/(tabs)/index.tsx`)
The recording dashboard relies on the native hardware bridges of `expo-av` to stream live microphone decibels to the **WaveformVisualizer**.
To prevent Android `cache` sandbox sweeps and `Maximum update depth` React collisions:
* Decibel metering is strictly throttled to **250ms**.
* Upon stopping a recording, the volatile `.m4a` buffer is natively cloned into `FileSystem.documentDirectory` to guarantee persistence across route transitions.
* The recording string and its measured `duration` interval are strictly injected into Expo Router URLs (`/note/new-note?audioUri=...&duration=...`) for the analysis engine to ingest.

### 2. The Gemini 2.5 Flash Translation Matrix (`app/note/[id].tsx`)
Rather than relying on outdated `whisper` logic or text-only NLP layers, the app maps the persistent `.m4a` audio directly into `Base64` chunks and feeds it into the strict multimodal engine of **Gemini 2.5 Flash**.

**Prompt Engineering Schema:**
The model is strictly commanded to evaluate the recording context and yield an explicit syntax output:
1. `TITLE: ` (A dynamically derived 3-5 word title)
2. `TOPIC: ` (A strict 1-2 word structural category)
3. `SUMMARY: ` (Detailed analytical markdown content)
4. `TRANSCRIPT: ` (Raw speech-to-text string)

A JavaScript formatting block regex-splits this output, computing realistic word counts and executing state injections to organically populate the UI layout parameters.

### 3. Native Markdown Display Engine
The unformatted string payload generated by Gemini is injected into the highly stable `react-native-markdown-display` package. The wrapper is custom-styled with dynamic literal `hex` constants pointing to our visual Theme engine (supporting bullet points, tables, blockquotes, and code snippets) natively rendered.

### 4. Express.js & Prisma Postgres Backend
The backend utilizes an isolated Express server bound to Port `8080`.
The Prisma ORM is connected to a rigorous remote cloud-hosted **PostgreSQL** deployment managed on Prisma Cloud parameters. 
The Schema natively enforces the generated Note parameters:
```prisma
model Note {
  id         String   @id @default(uuid())
  title      String
  topic      String
  duration   String
  wordCount  Int
  summary    String
  transcript String
  audioUrl   String?
  createdAt  DateTime @default(now())
  updatedAt  DateTime @updatedAt
}
```

## 🚀 Setup & Deployment

### Booting the Frontend
To clear the bundle cache and start the Metro Server natively:
```bash
npx expo start -c
```

### Starting the Backend
From the root `/backend` directory, run the TypeScript Watch compiler and initialize Database synchronization:
```bash
npx prisma db push
npx tsx watch src/index.ts
```

### Generating an Android APK
The project has been modified with a custom `eas.json` schema engineered to force a standard `.apk` build layer instead of Google's `.aab` format.
Simply execute the preview directive:
```bash
eas build -p android --profile preview
```
Download the resulting `.apk` file directly to your hardware for manual execution.
